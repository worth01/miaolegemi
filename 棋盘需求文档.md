> 版本：v1.3 · 8列×60px · 2026-05-07 · 新增临时存放层

## 一、棋盘坐标系统

### 1.1 二维网格坐标

每张棋盘卡牌由三个值定义其位置：

```javascript
{
  col:    3,     // 列坐标 (0-indexed)
  row:    2,     // 行坐标 (0-indexed)
  layer:  1,     // 层级 (0=最底层, 越大越高)
}
```

### 1.2 层间偏移

上层网格相对下层偏移半格，产生交错堆叠效果：

```
层0（底层）:  [A] [B] [C] [D] [E] [F] [G] [H]
                 ╲ ╱
层1（中层）:    [ [I] [J] [K] [L] [M] [N] ]
                     ╲ ╱
层2（顶层）:      [ [O] [P] [Q] [R] ]
```

偏移量 = tileSize / 2

```
层0 卡牌像素位置:  (col * tileSize,          row * tileSize)
层1 卡牌像素位置:  (col * tileSize + 0.5*t,  row * tileSize + 0.5*t)
层2 卡牌像素位置:  (col * tileSize + 1.0*t,  row * tileSize + 1.0*t)
```

其中 `t` = 单个卡牌尺寸（tileSize）。

### 1.3 卡牌渲染尺寸

| 元素 | 尺寸 |
|------|------|
| 卡牌视觉区域 | 60×60 px（设计稿） |
| 卡牌点击区域 | 60×60 px |
| 卡牌间距 | 0 px（无缝拼接） |
| 层间偏移 | 30 px（半格） |

### 1.4 层数上限

移动端最多 **4层**（层0-3），5层及以上在手机屏幕上遮挡严重，暂不使用。

## 三、卡牌类型分配规则

### 3.1 类型池

| 关卡 | 可用卡牌类型 |
|------|------------|
| 第一关（3种） | 猫爪印、猫罐头、逗猫棒 |
| 第二关（9种） | 猫爪印、猫薄荷、毛线球、猫罐头、猫爬架、逗猫棒、猫铃铛、猫条、蝴蝶结 |
| 第三关（9种） | 同第二关 |

### 3.2 分配约束

每张卡牌类型必须满足以下全部约束：

| 约束 | 说明 |
|------|------|
| 3的倍数 | 同种卡牌总数为3的倍数（保证全部可三消） |
| 跨层分散 | 同组3张卡牌必须分布在至少2个不同层 |
| 同层不相邻 | 同层内同类型卡牌不相邻（至少间隔1格） |
| 顶层覆盖 | 最顶层必须出现所有可用类型的卡牌 |

```javascript
function assignTypes(cards, layerConfig, typePool) {
  const total = cards.length;

  // 1. 按层分组
  const layers = groupByLayer(cards);

  // 2. 先分配顶层：确保每种类型至少出现1张
  const topLayer = layers[getMaxLayer(layers)];
  assignTopLayer(topLayer, typePool);

  // 3. 按类型分组，每组3张，满足跨层分散约束
  const groups = Math.floor(total / 3);
  for (let g = 0; g < groups; g++) {
    const type = typePool[g % typePool.length];
    const slots = pickSlotsAcrossLayers(layers, 3); // 每组3张分布至少2层
    slots.forEach(s => s.type = type);
  }

  // 4. 同层内检查相邻约束，必要时交换位置
  enforceNoAdjacentSameType(cards);

  // 5. Fisher-Yates 打乱同一层内同类型的位置
  shuffleWithinConstraints(cards);
}
```

### 3.3 可玩性校验

分配完成后跑模拟，确认棋盘可达（全部卡牌可被消除）：

```javascript
function isSolvable(cards, slotCount = 7) {
  const remaining = cards.map(c => ({ ...c, removed: false }));
  const slots = new Array(slotCount).fill(null);

  // BFS 或贪心模拟最优消除顺序
  while (true) {
    const clickable = remaining.filter(c => !c.removed && isClickable(c, remaining));
    if (clickable.length === 0) break;

    // 贪心策略：优先匹配已放入收集槽的
    let picked = clickable.find(c => {
      const matchInSlots = findPartialMatch(slots, c.type);
      return matchInSlots.length >= 2;
    });

    if (!picked) {
      // 无匹配时选任意一张
      picked = clickable[0];
    }

    // 放入收集槽
    const slotIdx = slots.indexOf(null);
    if (slotIdx === -1) return false; // 槽满 → 不可达
    slots[slotIdx] = picked;
    picked.removed = true;

    // 检查三消
    const match = checkMatch(slots);
    if (match.match) {
      clearSlots(slots, match.start);
    }
  }

  return remaining.every(c => c.removed);
}

// 校验失败则重新生成
function generateSolvableBoard(levelConfig, seed) {
  let board;
  let attempts = 0;
  do {
    board = generateBoard(levelConfig, seed + attempts);
    attempts++;
  } while (!isSolvable(board.cards) && attempts < 50);

  return board;
}
```

## 四、阻挡/可点击判断

### 4.1 阻挡规则（面积覆盖阈值）

一张卡牌 **可点击** 的条件：所有上层卡牌对它的覆盖面积占比 ≤ 30%。

```javascript
const BLOCK_THRESHOLD = 0.3; // 覆盖30%即判定为被遮挡

function isBlocked(card, allCards) {
  const cardRect = getPixelRect(card);
  let totalOverlap = 0;
  for (const other of allCards) {
    if (other.layer > card.layer && !other.removed) {
      totalOverlap += calcOverlapArea(getPixelRect(other), cardRect);
    }
  }
  const cardArea = TILE_SIZE * TILE_SIZE;
  return (totalOverlap / cardArea) > BLOCK_THRESHOLD;
}

function isClickable(card, allCards) {
  return !isBlocked(card, allCards);
}

function getPixelRect(card) {
  const x = card.col * TILE_SIZE + card.offsetX;
  const y = card.row * TILE_SIZE + card.offsetY;
  return { left: x, top: y, right: x + TILE_SIZE, bottom: y + TILE_SIZE };
}

function calcOverlapArea(a, b) {
  const w = Math.max(0, Math.min(a.right, b.right) - Math.max(a.left, b.left));
  const h = Math.max(0, Math.min(a.bottom, b.bottom) - Math.max(a.top, b.top));
  return w * h;
}
```

### 4.2 视觉标记

| 状态 | 视觉表现 |
|------|---------|
| 可点击 | 正常亮度，无遮罩 |
| 被遮挡（覆盖>30%） | 降低亮度（opacity 0.4） |
| 点击被遮挡卡牌 | 轻微抖动 + 亮度闪一下（反馈"这里被挡住了"） |

## 五、临时存放层

### 5.1 基础规则

```
[临时1] [临时2] [临时3]
```

- 临时存放层位于棋盘和收集槽之间，共 **3 格**
- 玩家可将棋盘上的卡牌放入临时存放层暂存，不占用收集槽
- 临时存放层的卡牌可随时点击移入收集槽
- 临时存放层有独立的空位判断，不受收集槽三消逻辑影响
- 视觉样式：橙色边框，`rgba(255,165,0,.1)` 背景填充

### 5.2 交互逻辑

- 点击棋盘可点击卡牌 → 飞入临时存放层最左侧空位（优先于收集槽）
- 或直接飞入收集槽（可由玩家选择，具体交互待定）
- 临时存放层已满时，只能直接飞入收集槽
- 点击临时存放层中的卡牌 → 移入收集槽

## 六、收集槽系统

### 6.1 基础规则

```
[槽位1] [槽位2] [槽位3] [槽位4] [槽位5] [槽位6] [槽位7]
```

- 收集槽位于临时存放层底部，共 **7 格**（可通过被动技能扩展）
- 玩家点击棋盘可点击卡牌 → 卡牌飞入收集槽最左侧空位
- 收集槽中**连续3张同类型卡牌** → 三消 → 自动清除这3张
- 收集槽满（7格全部占用）且无法三消 → 失败

### 6.2 三消判断

每放入一张新卡牌到收集槽后，检查：

```javascript
function checkMatch(slots) {
  for (let i = 0; i <= slots.length - 3; i++) {
    if (slots[i] && slots[i+1] && slots[i+2] &&
        slots[i].type === slots[i+1].type &&
        slots[i].type === slots[i+2].type) {
      return { match: true, start: i };
    }
  }
  return { match: false };
}
```

三消后，匹配的3张卡牌消除，后续卡牌自动左移填补空位：

```
消除前: [A] [B] [C] [C] [A] [ ] [ ]
消除后: [A] [B] [A] [ ] [ ] [ ] [ ]
```

### 6.3 满槽预警

收集槽剩余 **2 格** 时，收集槽边框渐变为橙色；剩余 **1 格** 时变为红色闪烁。

### 6.4 被动技能影响

| 被动技能 | 效果 |
|---------|------|
| 黑猫 · 灵光 | 收集槽上限 7→9 格 |
| 布偶猫 · 粘住 | 收集槽满时可强行塞入1张（主动技能触发） |

## 七、分数与产出系统

### 7.1 分数计算

| 事件 | 分数 | 说明 |
|------|------|------|
| 每次三消 | +10分 | 基础分 |
| 连消奖励 | combo × 5分 | 两次消除间隔 < 2秒视为连消 |
| 完美通关 | +200分 | 通关时收集槽为空 |

```javascript
function calcScore(eliminationHistory) {
  let score = 0;
  for (const event of eliminationHistory) {
    score += 10;
    if (event.timeSinceLastElim < 2000) {
      score += event.combo * 5;
    }
  }
  if (collectionEmpty && allCleared) {
    score += 200;
  }
  return score;
}
```

## 九、胜负与重来

### 9.1 胜利条件

**棋盘上所有卡牌被消除。**

```javascript
function checkWin(allCards) {
  return allCards.every(card => card.removed);
}
```

### 9.2 失败条件

**收集槽满 且 无法形成三消。**

```javascript
function checkLose(slots) {
  const allFilled = slots.every(s => s !== null);
  const noMatch = !checkMatch(slots).match;
  return allFilled && noMatch;
}
```

### 9.3 重来机制

重来 = **保留棋盘状态，清空收集槽**。

不重新生成棋盘，保证每日同款地图的公平性。

```javascript
function retryGame(gameState) {
  gameState.collectionSlots = new Array(7).fill(null);
  // 棋盘状态不变
  // 鱼干产出累计保留
  // 分数归零
  return gameState;
}
```

### 9.4 被动技能影响

| 被动技能 | 效果 |
|---------|------|
| 三花猫 · 三色守护 | 失败后可重来1次 |
| 波斯猫 · 优雅步调 | 每局额外1次重来机会 |

## 十二、渲染层级（Z-index）

卡牌绘制顺序从下层到上层，保证遮挡关系正确：

```
绘制顺序: 层0 → 层1 → 层2 → 层3 → 临时存放层 → 收集槽 → UI
```

每层内按 `row` 从上到下绘制，同 row 按 `col` 从左到右。

## 十三、屏幕适配

棋盘区域根据屏幕尺寸自动缩放，保持所有卡牌可见。

棋盘边界通过逐层计算最右/最下像素得到（不能用层数×偏移简单累加）：

```javascript
function getBoardBounds(layers) {
  let rightmost = 0, bottommost = 0;
  layers.forEach(la => {
    rightmost  = Math.max(rightmost,  (la.col + la.cols) * TILE_SIZE + la.offsetX);
    bottommost = Math.max(bottommost, (la.row + la.rows) * TILE_SIZE + la.offsetY);
  });
  return { boardW: rightmost, boardH: bottommost };
}

function calculateBoardScale(canvasWidth, canvasHeight, layers) {
  const { boardW, boardH } = getBoardBounds(layers);

  const availableWidth  = canvasWidth - 20;
  const availableHeight = canvasHeight - UI_TOP - UI_BOTTOM - 20;

  return Math.min(
    availableWidth / boardW,
    availableHeight / boardH,
    1
  );
}
```

## 十五、v2 路线图：干扰卡机制（设计中，未实施）

> 状态：设计提案 · 待评审 · 暂未进入开发排期
> 关联：[server/prisma/seed.js](server/prisma/seed.js) 蓝猫/三花猫技能；[猫咪消消乐.html](猫咪消消乐.html) 棋盘逻辑

### 15.1 立项背景

v1.3 棋盘只有 14 种普通收集卡，**所有卡功能完全一致**，仅靠图案区分。但两只猫咪技能文案声称作用于"干扰卡"：

| 猫 | 技能 | 当前文案 | 当前实际行为（[猫咪消消乐.html:1237-1248](猫咪消消乐.html#L1237-L1248)） |
|---|---|---|---|
| 蓝猫 | 猫抓板（被动） | 每局自动移出1张干扰卡 | 随机移出最顶层1张普通卡 |
| 三花猫 | 蹦跶（主动） | 随机跳走2张干扰卡 | 随机跳走2张普通卡 |

文案与代码名实不符，玩家以为存在的"干扰卡"机制实际不存在。v2 引入真正的干扰卡，让这两个技能的价值兑现，同时为正式关/竞技关提供难度爬升空间。

### 15.2 设计原则

1. **节制**：v2 首版只引入 2 种干扰卡，先验证体验再考虑扩展
2. **可破解**：每种干扰卡都有明确清除路径，不允许出现死局
3. **关卡分层**：教学关（第一关）保持纯净，正式关（第二关+）开始引入
4. **可玩性校验兼容**：现有 `isSolvable` 模拟逻辑需扩展，加入干扰卡处理路径

### 15.3 干扰卡类型（v2 首版）

#### 15.3.1 毛球（占位型干扰）

| 项 | 内容 |
|---|---|
| 视觉 | 灰色绒毛球 + 红色"✕"标记，区别于普通卡的彩色道具图案 |
| 行为 | 占据棋盘位置，无法被点击进入收集槽 |
| 清除方式 | ① 蓝猫·猫抓板：每局自动清除1个最顶层毛球<br>② 三花猫·蹦跶：随机清除最多2个毛球<br>③ 自然清除：当其相邻4格全部被消除后，该毛球自动消失 |
| 生成规则 | 第二关：每局 4-6 个，禁止出现在最顶层<br>第三关：每局 8-10 个 |

#### 15.3.2 猫毛绳结（链接型干扰）

| 项 | 内容 |
|---|---|
| 视觉 | 2 张同层相邻卡之间渲染一根动态摆动的红色绳索（SVG path + sin 波动画） |
| 行为 | 被绳索连接的2张卡必须先清除其中一端才能解锁另一端，单击任一端无效 |
| 清除方式 | ① 长按或双击一端：先清除该端进入收集槽，另一端绳索消失<br>② 三花猫·蹦跶（升级版）：可强制解除1组绳结 |
| 生成规则 | 第二关：1-2 组<br>第三关：2-3 组<br>仅在层 0 和层 1 之间出现，避免上层绳索视觉混乱 |

### 15.4 与现有技能联动重写

| 猫 | v1 描述 | v2 描述 |
|---|---|---|
| 蓝猫·猫抓板（被动） | 每局自动移出1张干扰卡 | 每局自动清除1个最顶层毛球；无毛球时改为清除1张顶层普通卡 |
| 三花猫·蹦跶（主动） | 随机跳走2张干扰卡 | 优先清除2个毛球，毛球不足时清除剩余绳结（最多1组） |

> 同步更新 [server/prisma/seed.js](server/prisma/seed.js) 的 `activeSkill` / `passiveSkill` 字段。

### 15.5 视觉与动画规范

| 事件 | 动画 | 时长 |
|---|---|---|
| 毛球进场 | 从空中飘落 + 弹性着陆 + 灰色绒毛粒子 | 0.4s |
| 毛球被技能清除 | 缩小至 0 + 飞散为绒毛粒子 | 0.5s |
| 毛球自然消失（相邻全清） | 渐隐 + 一声"喵～" | 0.3s |
| 绳结生成 | 两端卡牌之间画出绳索，sin 波摆动 | 持续 |
| 绳结解除 | 绳索从中点断开 + 弹性回弹 + 飞散 | 0.4s |

### 15.6 可玩性校验扩展

现有 [isSolvable](棋盘需求文档.md#L185) 在贪心模拟时需要扩展规则：

```javascript
function isSolvable(cards, slotCount, interferences) {
  // 1. 干扰卡不进入收集槽，但参与遮挡判断
  // 2. 毛球：当且仅当其相邻4格全清后，可被自然清除
  // 3. 绳结：两端必须按"先A后B"或"先B后A"顺序清除
  // 4. 开局必须至少存在 N 张可点击的非干扰卡（避免立即死局）
  // 5. 模拟需把"使用主动技能（蹦跶/灼烧等）"作为可选动作纳入搜索空间
}
```

校验失败时自动重新生成棋盘（沿用 v1.3 最多50次重试机制）。

### 15.8 验收标准

- [ ] 棋盘生成时按规则正确放置毛球与绳结
- [ ] 蓝猫·猫抓板、三花猫·蹦跶 在有干扰卡时优先作用于干扰卡
- [ ] 引入干扰卡后 `isSolvable` 校验通过率 ≥ 90%（重试 ≤ 50 次）
- [ ] 视觉清晰区分干扰卡 vs 普通卡（色彩、边框、icon、动画）
- [ ] 干扰卡清除有明显多通道反馈（音效 + 粒子 + 文字提示）
- [ ] 第一关无干扰卡，保留作为纯净学习场景
- [ ] 干扰卡相关文案在所有展示位置（卡牌详情/结算页/技能描述）保持一致
