## P8: 观战猫咪被动技能通知横幅

### 设计目标
在消消乐游戏中，顶部状态栏下方显示观战猫咪的被动技能效果提示，让玩家直观感受到猫咪带来的增益。

### 实现位置
- 文件：`猫咪消消乐.html`
- 位置：顶部进度条下方，棋盘上方

### 显示内容

| 猫咪位置 | 被动技能 | 横幅示例 |
|---------|---------|---------|
| 4号位 | 星空猫·观星 | 🛡️星空猫：时刻祝福你，增加幸运触发 |
| 5号位 | 布偶猫·贵宾 | 🛡️布偶猫：鱼干+20% |
| 6号位 | 无毛猫·爪痕 | 🛡️无毛猫：每局自动移出1张干扰卡 |

### UI样式
- 紫色半透明背景
- 紫色边框
- 平滑淡入动画
- 每个被动技能单独一行

### 触发条件
- 观赛区4号位（被动位）猫咪存在时显示
- 观赛区5-6位猫咪存在时显示
- 空位不显示

## P7: 抽卡保底系统

### 设计目标
提升玩家抽卡体验，避免长时间不出高稀有度的"长草期"，同时保持抽卡的惊喜感。

### 双轨保底机制

| 保底类型 | 触发条件 | 保底内容 | 说明 |
|---------|---------|---------|------|
| 💎 **SR小保底** | 每20抽 | 至少SR | SR小保底有30%概率升为SSR |
| ✨ **SSR大保底** | 每80抽 | 必出SSR | 保底触发后计数器重置 |

### SSR概率递进表

| 累计抽数 | SSR概率 | 说明 |
|---------|--------|------|
| 0-19抽 | 2% | 基础概率 |
| 20-39抽 | 5% | 第1次递进 |
| 40-59抽 | 10% | 第2次递进 |
| 60-79抽 | 20% | 第3次递进 |
| 80抽 | 100% | 保底必出 |

### 基础概率分布（不含保底）

| 稀有度 | 概率 | 说明 |
|-------|------|------|
| N | 75% | 普通 |
| R | 18% | 稀有 |
| SR | 5% | 精良 |
| SSR | 2% | 传说 |

### UI展示

- **双进度条**：分别显示SR小保底(20抽)和SSR大保底(80抽)进度
- **概率提示**：显示当前抽数的SSR概率
- **保底预警**：快触发保底时显示火焰图标和倒计时
- **结果弹窗**：触发保底时显示特殊动画和标签

## 一、游戏定位

### 核心逻辑

```
猫是游戏引擎，不是收集终点。
消消乐是每日互动方式，不是主线。
```

玩家每天的循环：
```
打消消乐 → 获得鱼干 → 决定鱼干用途 → 管理猫咪阵容
```

### 设计原则

- **零压力回归**：长时间离线后回来，家园的猫依然在等你
- **决策有重量但不焦虑**：鱼干不够时需要做取舍，但不会丢失核心资产
- **变化来自猫咪阵容**：不需要大量关卡，不同猫组合产生不同体验
- **猫咪有双重价值**：编号唯一性 + 技能实用性，缺一不可

## 二、猫咪系统

### 2.1 猫咪种类（12种）

| 稀有度 | 猫咪 | 数量 | 权重 |
|--------|------|------|------|
| N | 橘猫、黑猫、白猫、蓝猫 | 4 | 30 |
| R | 布偶猫、三花猫、暹罗猫、无毛猫 | 4 | 15 |
| SR | 波斯猫、折耳猫 | 2 | 5 |
| SSR | 星空猫、幸运猫 | 2 | 5 |

每只猫有两个维度：
- **主动技能**：放在出战席 1-3 位时触发，对应消消乐底部按钮
- **被动技能**：放在观赛区 4 位或观赛区 5-6 位时触发，全局生效

| 白猫 | N | **醒目** | 每局开局立即消除1组可消除组合 |
| 蓝猫 | N | **沉静** | 鱼干+10% |
| 布偶猫 | R | **贵宾** | 鱼干+20% |
| 三花猫 | R | **收获** | 临时存放层槽位上限+2格 |
| 暹罗猫 | R | **吃得快** | 每局计时从开局5秒后开始计算 |

### 2.3 猫咪性格

每只猫抽卡时随机继承 2 个性格词（终身不变）：

```
傲娇、吃货、胆小、黏人、高冷、话痨、懒散、胆大、独行、黏糊、
```

性格词显示在卡面，是社交炫耀维度。

### 2.4 序列号系统

- 每只猫有全局唯一编号（例如 `橘猫 · #00847`）
- 发行时分三个批次：
  - **创世区**（#001-100）：永久特殊标记，通过活动释放
  - **早期区**（#101-500）：首月进入抽卡池
  - **标准区**（#501+）：随玩家增长自然增发
- 初次发现者永久记录在卡面上

## 三、猫咪三区机制

猫咪系统包含三个区：**备战区**、**家园** 和 **包裹区**。

### 3.1 包裹区

- 抽卡后猫咪默认进入包裹区
- **30天倒计时**，不可延长
- 到期后猫咪自动流浪，序列号进入60天冷却 → 回到池子
- 容量上限 20 格，可用鱼干扩容

### 3.2 家园（亲密度系统）

- 从包裹区领养 / 从出战席撤回 / 流浪救回，猫进入家园
- **永生**：猫咪永远不死、不流浪
- 初始容量10格，通过鱼干解锁，150条鱼干解锁一格
- 猫咪生活在「木头货架」风格的家园中，显示亲密度进度条

**亲密度机制：**

| 参数 | 值 |
|------|-----|
| 亲密度上限 | 100 |
| 喂1次 | +5 亲密度 |
| 在线衰减 | 每自然日 -1 |
| 离线衰减 | **暂停**（未登录的时段不计入衰减） |
| 加成映射 | 亲密度 0%→100%，技能加成线性对应，无悬崖 |

亲密度影响：
- **主动技能**：加成倍率随亲密度变化
- **被动技能**：效果强度随亲密度变化
- 亲密度 = 0 时：猫仍在，技能加成 = 0%，身份/血统/纪念全保留

**喂食 UI：**
- 只显示一条爱心进度条（❤️ 85/100）
- packing / starving / hungry 状态全部移除

### 3.3 出战席

- 从家园拖动猫咪上场至出战席，固定 **3个槽位**（不扩容）
- 前 3 格 = **主动技能位**，对应消消乐底部 3 个技能按钮
- 第 4 格 = **被动技能位**，全局生效
- 出战席和家园之间自由移动，**零成本、零等待**
- 猫咪出战期间，亲密度衰减照常运行，喂食需回家园操作

**出战席视觉：** 暗金大理石圆形站台，1-3号位编号，显示主动/被动标签。

### 3.4 猫咪不和机制

部分猫咪之间存在排斥，**不能同时出战**：

| 冲突对 | 原因 |
|--------|------|
| 黑猫 + 三花猫 | 灵光(+1临时槽) 与 收获(+2临时槽) 冲突 |

> 若玩家尝试同时上阵冲突对，系统提示「这两只猫性格不合，不能同时出战」并阻止。

### 3.5 家园 → 流浪（仅由主动弃养触发）

猫从家园离开只能去流浪，不能回到包裹区。安全网：

```
点击"送走" → 告别仪式（24小时打包状态）
  → 猫咪在出战席/家园显示背着小背包
  → 24小时内：
    点击"还是留下你" → 花费鱼干 → 猫咪卸下背包回来
  → 24小时后：
    猫咪真正流浪 → 不可逆 → 序列号回到池子
```

### 3.6 流浪 → 救回

- 救回的猫直接进入**家园**，不再回包裹区
- 亲密度重置为 0，需要重新喂养恢复加成

### 3.7 三区完整状态流

```
抽卡 → 包裹区（30天）
              ├── 领养 → 家园（永生，亲密度系统）
              │           ├── 拖动上场 →出战席（1-3号位，主动技能） 观战区（4-6号位，猫咪被动技能生效）
              │           ├── 撤回 ← 出战席/观战区（零成本）
              │           └── 送走 → 24h告别（鱼干可挽回）→ 流浪 → 回池子
              └── 30天到期 → 自动流浪 → 60天冷却 → 回池子
流浪 → 救回 → 直接回家园（亲密度=0）
```

## 四、消消乐系统

### 4.1 棋盘卡牌

消消乐棋盘使用 9 种猫咪相关物品卡牌（非猫咪本身）：

```
猫爪印、猫薄荷、毛线球、猫罐头、猫爬架、逗猫棒、猫铃铛、猫条、蝴蝶结
```

- 这些卡牌只在消消乐棋盘出现
- 第一关固定猫爪印，猫罐头，逗猫棒三种
- 第二关9种类型
- 不在抽卡区，不能被收集
- 每日棋盘全服同一张（算法生成 seed）

### 4.2 出战阵容

消消乐底部 3 个按钮 = 出战席前 3 位猫咪的主动技能。
观战区4-6位猫咪的被动技能,同时生效（全局叠加）。

```
出战席槽位：
  [1号位] [2号位] [3号位]  →  消消乐底部3个主动技能按钮
```
```
观战区槽位：
  [4号位] [5号位] [6号位]  →   被动加成（全局生效），消消乐顶部消息提示
```

不同阵容 = 不同策略：
```
追求速度通关 → 橘猫主动+无毛猫被动+暹罗猫被动
追求鱼干收益 → 布偶猫被动+幸运猫被动+蓝猫被动
追求容错率   → 三花猫被动+波斯猫被动+黑猫主动
追求掌控感   → 折耳猫主动+暹罗猫主动+白猫主动
```

**猫咪不和：** 黑猫与三花猫不能同时出战，系统自动阻止。

### 4.3 双轨收益

```
随便消几下 → 掉落鱼干（养成资源）     ← 休闲玩家
拼命打高分 → 积分上周榜/解锁称号      ← 竞技玩家
```

## 五、经济系统

### 5.1 双货币

| 货币 | 获取 | 用途 |
|------|------|------|
| 鱼干 | 消消乐产出、每日签到、周榜奖励 | 包裹区扩容、家园扩容、消消乐道具、抽卡辅助、喂食、挽留送走的猫 |
| 猫铃铛 | 成就里程碑、满勤奖励、周榜Top N | 抽卡（核心门票） |

### 5.2 鱼干用途明细

| 用途 | 消耗 | 意义 |
|------|------|------|
| 包裹区扩容 | 500 / 1200 / 2500 鱼干 | 扩到 20→30→40→50 格 |
| 家园扩容 | 150 / 300 / 450 鱼干 | 扩到 10→11→12 格 |
| 喂食 | 1 鱼干/次 | +5 亲密度 |
| 批量喂食 | 按需扣鱼干 | 一键补满亲密度 |
| 消消乐道具 | 10 鱼干/局 | 本局多给1次道具 |
| 抽卡辅助 | 10 鱼干/次 | 提高指定稀有度概率 |
| 告别挽留 | 递增1-5鱼干/天 | 送走的猫24h内可追回 |

### 5.3 经济闭环

```
消消乐（免费玩）→ 产出鱼干
                     ↓
           鱼干 → 扩容/道具/抽卡辅助/挽留
                     ↓
成就/满勤 → 产出猫铃铛（稀缺）
                     ↓
           猫铃铛 → 抽卡 → 包裹区 → 领养进家园
```

## 八、后端技术架构

### 8.1 技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| 数据库 | PostgreSQL | 序列号并发需要行锁 |
| ORM | Prisma | 类型安全，迁移管理 |
| 后端框架 | Node.js + Express | 与前端同语言 |
| 认证 | JWT | 无状态 |

### 8.2 核心数据库表

#### users（用户）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| username | VARCHAR | 账号 |
| nickname | VARCHAR | 随机猫名 |
| pity_count | INTEGER | 保底计数 |

#### cat_species（猫咪种类，12种静态数据）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL | 主键 |
| name | VARCHAR | 猫咪名称 |
| rarity | VARCHAR | N/R/SR/SSR |
| active_skill | JSONB | 主动技能描述 |
| passive_skill | JSONB | 被动技能描述 |
| weight | INTEGER | 抽卡权重 |

#### cat_serial_registry（序列号注册表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| species_id | INTEGER | 猫种类 |
| serial_number | INTEGER | 编号（种类内唯一） |
| status | VARCHAR | available/adopted/wandering/cooling |
| current_owner_id | UUID | 当前主人 |
| first_owner_id | UUID | 初次发现者（永久记录） |
| available_after | TIMESTAMPTZ | 冷却结束时间 |

#### player_cats（玩家持有的猫）
| 字段 | 类型 | 说明 |
|------|------|------|

### 8.4 关键技术决策

**鱼干用账本，不用余额字段**
防止篡改，方便回溯。

**序列号分配用数据库行锁**
```sql
SELECT * FROM cat_serial_registry
WHERE species_id = ? AND status = 'available'
ORDER BY serial_number ASC
LIMIT 1
FOR UPDATE SKIP LOCKED
```

## 十、核心设计理念

> **消消乐上的卡牌是任务，出战席的猫是工具。**
> **包裹区是限时决策，家园是永久安全。**
> **鱼干不续命，但能补亲密。**
> **召回友好性 > 日活驱动力。**

不是在做一个消消乐加上收藏系统，而是在做一个**猫咪阵容养成 × 消除游戏**，消消乐是展示猫咪价值的舞台，猫咪是驱动消消乐策略的核心。

### 交互规则

#### 拖拽上阵

| 操作 | 触发条件 | 结果 |
|------|---------|------|
| 向上拖拽 | 滑动距离 > 40px | 优先进入出战区空位 |
| 向下拖拽 | 滑动距离 > 40px | 优先进入观战区已解锁空位 |
| 区域已满 | 无空位 | 卡牌回弹动画 + 文字提示 |
| 锁定槽位 | 目标为锁定位 | 拒绝放入 + 提示"观战位未解锁" |

#### 点击撤回
- 点击出战区 / 观战区内猫咪 → 返回家园卡牌栏
- 对应卡槽恢复「空位」状态
- 家园猫咪恢复可拖拽状态

#### 卡牌状态

| 状态 | 外观 | 可操作性 |
|------|------|---------|
| 未上阵 | 彩色渐变、正常显示 | 可拖拽、可点击 |
| 已上阵 | 灰色半透明 | 不可操作 |
| 拖拽中 | 放大+阴影跟手 | — |

#### 卡牌栏横向滑动
- 支持手指横滑 / 鼠标滚轮横向滚动
- 拖拽与横向滑动互不冲突（通过移动距离方向判断优先级）
- 滚动流畅无卡顿，隐藏滚动条
