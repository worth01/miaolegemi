# 喵了个咪 · 页面开发需求文档

> 基于猫咪养成循环系统设计，版本 v2.0

---

## 一、页面结构总览

```
底部导航栏（4个Tab）
├── 首页        主页大厅，养成区快览 + 通知
├── 消消乐      消除游戏，产出鱼干
├── 猫咪        养成区 + 包裹区（核心新增）
└── 我的        个人中心，称号/里程碑/纪念册
```

---

## 二、全局组件

### 顶部货币栏（所有页面常驻）

```
[鱼干图标] 128    [猫铃铛图标] 3
```

- 鱼干：日常高频资源，喂猫消耗
- 猫铃铛：稀缺资源，抽卡专用，整数显示
- 点击任一货币 → 弹出来源说明浮层

### 通知横幅（首页 + 猫咪页）

优先级从高到低，同时只显示最高优先级一条：

| 优先级 | 触发条件 | 文案 |
|--------|---------|------|
| 1 | 养成区有猫处于 packing 状态 | "橘猫 #234 正在收拾行李，还有24小时" |
| 2 | 养成区有猫处于 starving 状态 | "黑猫 #891 已经4天没吃东西了" |
| 3 | 包裹区猫咪剩余 ≤ 7 天 | "包裹区有1只猫还有7天就会离开" |
| 4 | 养成区有猫处于 hungry 状态 | "有猫咪需要喂食" |

点击横幅 → 跳转到对应猫咪详情页

## 三、首页（Tab 1）

### 3.1 布局结构

```
┌─────────────────────────────┐
│  [头像] 昵称  称号标签       │  ← 顶部用户信息
│  [通知横幅]                  │  ← 条件显示
├─────────────────────────────┤
│                             │
│      养成区预览区            │  ← 主要区域
│  [猫1][猫2][猫3][+][+][+]  │
│                             │
├─────────────────────────────┤
│  [签到]  今日任务进度        │  ← 每日区域
├─────────────────────────────┤
│  消消乐入口                  │  ← 快捷入口
│  "今日已获得 12 条鱼干"      │
└─────────────────────────────┘
```

### 3.2 养成区预览区

- 显示全部槽位（初始3格，最多6格）
- 每个已占用槽位显示：
  - 猫咪卡面（含稀有度边框颜色）
  - 状态图标：🟢正常 / 🟡饥饿 / 🔴快离开
  - 剩余天数：`还有 12 天`
- 空槽显示 `+` 虚线框，点击 → 跳转包裹区
- 点击任意猫咪卡 → 跳转猫咪详情页

**状态图标规则：**

| 状态 | 图标颜色 | 附加动效 |
|------|---------|---------|
| alive | 绿色实心圆 | 无 |
| hungry | 黄色实心圆 | 轻微抖动 |
| starving | 橙色实心圆 | 快速抖动 |
| packing | 红色闪烁 | 持续闪烁 |

### 3.3 签到区域

- 今日是否已签到状态
- 签到奖励：鱼干 × 5（连续7天：第7天额外给猫铃铛 × 1）
- 点击签到 → 动画播放 → 货币增加

### 3.4 消消乐快捷入口

- 显示今日已通过消消乐获得的鱼干数
- 按钮文案：`去打消消乐` / `今日继续`
- 点击 → 跳转消消乐页

## 四、猫咪页（Tab 3）

### 4.1 布局结构

```
┌─────────────────────────────┐
│  [养成区] [包裹区]           │  ← Tab 切换
├─────────────────────────────┤
│                             │
│   （养成区 / 包裹区内容）     │
│                             │
└─────────────────────────────┘
```

### 4.2 养成区 Tab

**槽位卡片（每张卡）：**

```
┌──────────────────┐
│  [猫咪图片]       │
│  星空猫           │
│  #0023  SSR      │
│  ●●●○○  饥饿中   │  ← 状态
│  还有 8 天        │  ← 倒计时
│  [喂食 -1鱼干]   │  ← 操作按钮
└──────────────────┘
```

- 喂食按钮点击：
  - 鱼干 ≥ 1：扣除1条，expires_at +1天，状态回到 alive，播放喂食动画
  - 鱼干 = 0：按钮置灰，提示"鱼干不足，去打消消乐获得"

**空槽：**

```
┌──────────────────┐
│                  │
│    + 领养新猫    │
│  包裹区有2只等你  │  ← 有猫时显示数量
└──────────────────┘
```

点击 → 跳转包裹区

**里程碑解锁提示（第4/5/6格）：**

若槽位未解锁，显示灰色锁定状态：

```
┌──────────────────┐
│     🔒           │
│  领养10只后解锁   │
│  当前 3/10       │
└──────────────────┘
```

### 4.3 包裹区 Tab

**列表项（每只猫）：**

```
┌─────────────────────────────────┐
│ [猫图] 橘猫  #0234  N           │
│        性格：傲娇 · 吃货         │
│        ⏰ 还有 23 天会离开       │
│                    [领养]        │
└─────────────────────────────────┘
```

**点击领养按钮逻辑：**

- 养成区有空槽 → 确认弹窗：
  ```
  "正式领养这只橘猫 #0234？
   它将开始在你这里生活，需要每天喂食。"
  [取消]  [领养它]
  ```
  确认后：播放领养动画，猫移入养成区，开始30天计时

- 养成区已满 → 提示弹窗：
  ```
  "养成区已满（3/3）
   需要先送走一只猫才能领养新成员。"
  [去养成区]  [取消]
  ```

**空状态：**

```
包裹区空空如也
去抽卡带回一只猫吧
[去抽卡]
```

## 九、核心交互流程

### 流程 A：每日循环

```
打开App
  → 查看首页通知横幅（是否有猫需要喂食/包裹区到期）
  → 点击签到（获得鱼干）
  → 打一局消消乐（获得鱼干）
  → 去养成区喂猫（消耗鱼干）
  → 查看包裹区（决定是否领养新成员）
  → 关闭App
```

### 流程 B：抽卡到领养

```
用猫铃铛抽卡
  → 猫进入包裹区（30天计时开始）
  → 收到通知（还有7天到期）
  → 打开包裹区
  → 养成区有空槽 → 点击领养 → 确认 → 猫进养成区
  → 养成区已满 → 决定送走一只 → 腾出槽位 → 领养
```

### 流程 C：猫咪去猫星

```
猫到期未喂食 → 状态变 packing
  → 首页通知横幅显示（最高优先级）
  → 玩家24小时内喂食 → 续命
  OR
  → 24小时内未处理 → 后端自动处理离开
  → 猫进入纪念册
  → 序列号进入60天冷却
  → 槽位释放
```

### 流程 D：里程碑解锁第4格

```
历史领养达到10只
  → 系统弹出通知：「养成区第4格已解锁！」
  → 养成区新增一个空槽
```

## 十、Push 通知需求

| 触发时机 | 通知文案 |
|---------|---------|
| 猫咪进入 hungry 状态（2天未喂） | "橘猫#234 饿了，快去喂它一点鱼干" |
| 猫咪进入 starving 状态（4天未喂） | "橘猫#234 已经4天没吃东西了，它还在等你" |
| 猫咪进入 packing 状态（最后24小时） | "橘猫#234 开始收拾行李了，还有最后24小时" |
| 包裹区猫咪剩余7天 | "包裹里有只猫还有7天就要离开了" |
| 签到提醒（每天10:00） | "今天还没签到，顺手喂猫了吗？" |

## 十一、数据接口需求（前端视角）

前端需要以下接口支持页面渲染：

```
GET  /cats/nurturing            养成区猫咪列表（含状态/剩余天数/加成）
GET  /cats/bag                  包裹区猫咪列表（含剩余天数）
POST /cats/:id/feed             喂食（消耗1鱼干，+1天）
POST /cats/:id/adopt            领养（包裹区→养成区）
POST /cats/:id/release          主动送去猫星
GET  /cats/:id/detail           猫咪详情（含成长档案）
GET  /cats/:serialId/lineage    序列号血统记录

GET  /game/today-board          今日棋盘seed + 加成列表
POST /game/finish               提交本局结果（返回鱼干发放数）

GET  /user/milestones           里程碑进度
GET  /user/memorial             纪念册列表
GET  /user/stats                个人统计数据
```
